# QA Test Plan - Alpha V1.0

본 문서는 **Big History AI Platform Alpha V1.0**의 기능 및 품질 검증을 위한 테스트 시나리오를 정의합니다.

## 1. 테스트 개요
- **버전**: Alpha V1.0
- **테스트 환경**: Chrome (Latest), Resolution 1920x1080 (Desktop)
- **목표**: 주요 기능(챗봇, 강의 생성, 인쇄)의 정상 작동 여부 및 디자인 정합성 검증

## 2. 기능별 테스트 시나리오

### 2.1 챗봇(Q&A) 모드
| ID | 테스트 항목 | 절차 | 예상 결과 | 비고 |
|:---:|:---|:---|:---|:---|
| QA-01 | 일반 질문 응답 | 1. `Q&A Mode` 선택<br>2. "빅히스토리란 무엇인가?" 입력 후 전송 | - 답변이 스트리밍으로 생성됨<br>- 출처(Citations)가 함께 표시됨 | |
| QA-02 | 관련 없는 질문 처리 | 1. 빅히스토리와 무관한 질문 입력 (예: "라면 끓이는 법") | - 정중하게 답변을 거절하거나 데이터 범위 밖임을 안내 | |

### 2.2 강의 생성(Curriculum) 모드
| ID | 테스트 항목 | 절차 | 예상 결과 | 비고 |
|:---:|:---|:---|:---|:---|
| CUR-01 | 학년 선택 필수 | 1. `Curriculum Generation Mode` 선택<br>2. 학년 선택 없이 질문 입력 | - "강의 대상을 선택해주세요" 알림(Alert) 발생 | |
| CUR-02 | 강의 노트 생성 | 1. 학년(예: 초등 고학년) 선택<br>2. 주제(예: "우주의 탄생") 입력 | - **Textbook 스타일**의 강의 노트 생성<br>- 제목, 본문, 핵심 질문 등이 명조체로 깔끔하게 표시 | |
| CUR-03 | 답변 박스 인터랙션 | 1. 생성된 강의 노트 활성화(Canvas 열기)<br>2. "핵심 질문" 섹션의 `[답변]` 버튼 클릭 | - **버튼이 사라짐**<br>- 질문 하단에 **회색 박스**가 생기며 상세 답변 표시<br>- 답변 폰트가 제목 서체(Outfit)로 변경됨 | **신규 기능** |

### 2.3 라이브러리 및 도구
| ID | 테스트 항목 | 절차 | 예상 결과 | 비고 |
|:---:|:---|:---|:---|:---|
| LIB-01 | 저장 기능 | 1. 생성된 답변 하단의 `[라이브러리에 저장]` 클릭<br>2. 우측 상단 햄버거 메뉴(`≡`) 클릭 | - 라이브러리 사이드바 열림<br>- 방금 저장한 항목이 최상단에 표시됨 | |
| PDF-01 | PDF 내보내기 | 1. CanvasPanel 우측 상단 `[프린터 아이콘]` 클릭<br>2. 인쇄 미리보기 확인 | - **A4 용지 마진**이 적용됨<br>- 불필요한 버튼(닫기, 저장 등)이 보이지 않음<br>- 본문 내용만 깔끔하게 출력 | **신규 기능** |
| EDT-01 | 편집 모드 | 1. CanvasPanel 우측 상단 `[연필 아이콘]` 클릭<br>2. 내용 수정 후 다시 클릭하여 저장 | - 텍스트 수정 가능<br>- 수정 후 뷰어 모드로 돌아왔을 때 변경 내용 반영 | |

### 2.4 디자인(UI/UX)
| ID | 테스트 항목 | 절차 | 예상 결과 | 비고 |
|:---:|:---|:---|:---|:---|
| UI-01 | 폰트 적용 | 1. 전체 페이지 폰트 확인 | - 본문: `Noto Serif KR` (명조체) 적용 확인<br>- 가독성이 좋은 줄 간격(1.6~1.8) 확인 | |
| UI-02 | 공무서 스타일 제거 | 1. 강의 노트 생성 결과 확인 | - `1., 가., 1)` 형태의 강제된 위계 없음<br>- 자연스러운 줄글 형태 확인 | |
