# 멀티모달 이미지 통합 규칙 (Image Integration Rules)

이 문서는 AI가 강의 자료를 생성할 때, 추출된 PDF 이미지(삽화, 도표)를 본문에 배치하는 기준과 비율을 정의합니다.

## 1. 학년별 이미지 배치 비율 (Image Density)

대상 학년(Target Audience)에 따라 텍스트 대비 이미지의 비율을 다르게 설정합니다.

| 대상 (Target) | 이미지 비율 | 배치 가이드 |
| :--- | :--- | :--- |
| **초등학교 (Elementary)** | **높음 (High)** | - **문단 1~2개당 이미지 1개** 권장<br>- 추상적인 개념보다는 구체적인 사물/현상 이미지 위주<br>- 흥미 유발을 위해 핵심 질문(Key Questions) 옆에 필수 배치 |
| **중학교 (Middle School)** | **중간 (Medium)** | - **각 섹션(Section)마다 최소 1개** 배치<br>- 개념 설명(Core Concepts)을 보조하는 도식/그래프 위주<br>- 텍스트 흐름을 끊지 않는 선에서 배치 |
| **고등학교/성인 (High/Adult)** | **낮음 (Low)** | - **필요한 경우에만** (증거 자료, 복잡한 데이터 시각화)<br>- 단순 삽화보다는 정보 전달용 도표 위주 |

## 2. 이미지 적합성 판단 기준 (Suitability Criteria)

AI는 다음 기준에 따라 이미지를 선택하거나 배치를 결정해야 합니다.

*   **관련성 (Relevance)**: 텍스트 내용과 직접적으로 관련된 이미지만 사용합니다. (장식용 장미꽃 X, 주제와 무관한 패턴 X)
*   **설명력 (Explanatiory Power)**: "백문이 불여일견"인 경우 우선 배치합니다. (예: 우주의 팽창 모델, DNA 구조 등 글로 설명하기 어려운 것)
*   **위치 선정 (Placement)**:
    *   **설명 직후**: 개념을 설명한 문장 바로 다음에 배치하여 이해를 돕습니다.
    *   **활동 예시**: 학생 활동(Student Activities) 섹션에서 예시 자료로 활용합니다.

## 3. 기술적 구현 규칙 (Automation Rules)

*   **이미지 소스**: `gs://20set-bighistory-raw/extracted_images/` (또는 해당 Public URL)
*   **파일명 매칭**: RAG 검색 결과의 출처 페이지(Page Number)와 이미지 파일명의 페이지 번호(`_p024_`)가 일치하는 이미지를 후보로 선정합니다.
*   **Markdown 삽입 포맷**:
    ```markdown
    ![이미지 설명](이미지URL)
    *그림 1: [이미지에 대한 짧은 캡션]*
    ```

## 4. 커스텀 설정 (User Overrides)

특정 키워드에 대해 강제로 이미지를 넣거나 뺄 수 있습니다.

*   **[FORCE_IMAGE]**: "빅뱅", "지구 탄생", "공룡" 키워드가 나오면 반드시 관련 이미지를 1장 이상 찾아서 넣으세요.
*   **[NO_IMAGE]**: "참고 문헌", "개요" 섹션에는 이미지를 넣지 마세요.
